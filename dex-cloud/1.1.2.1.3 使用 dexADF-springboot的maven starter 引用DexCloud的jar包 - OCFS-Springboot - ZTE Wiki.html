1.1.2.1.3 使用 dexADF-springboot的maven starter 引用DexCloud的jar包

转至元数据结尾
  • 由 胡锐10071214创建, 最后修改于十一月 22, 2017
转至元数据起始

       依赖管理几乎是所有复杂项目的关键部分。以手动的方式来实现依赖管理不太现实，你得花更多时间，同时你在项目的其他重要方面能付出的时间就会变得越少。springboot为了解决这个问题提出了Spring Boot starter的方案，Starter POM 是一组方便的依赖描述符，您可以将其包含在应用程序中。您可以获得所需的所有 Spring 和相关技术的一站式服务，无需通过示例代码搜索和复制粘贴依赖。

    springboot 的starter机制可以参考：9.1.1.1 Spring Boot Starter 介绍 和 9.1.1.2 SpringBoot四大神器之Starter

    而dexcloud中目前所提供的starter如下：

dexADF-springboot starter的maven坐标（其groupId都为com.zte.oes.dexcloud，其version跟随dexcloud版本变化）
相关功能介绍
dexADF-springboot starter的maven坐标（其groupId都为com.zte.oes.dexcloud，其version跟随dexcloud版本变化）
相关功能介绍
dexcloud-springboot-starterdexADF-springboot的基础starter，其中包含了 dexcloud-springboot-starter-commons和 dexcloud-springboot-starter-msb
dexcloud-springboot-starter-msb提供了与msb进行连接，查询，注册相关功能
dexcloud-springboot-starter-jetty提供使用jetty作为web容器，在springboot对jetty的基础上，提供了更多的配置参数的支持，如：线程池数量
dexcloud-springboot-starter-jersey提供了使用jersey框架作为restful的开发框架，并且提供的对swagger的集成支撑
dexcloud-springboot-starter-rpc-retrofit提供了使用retrofit作为微服务之间点对点rpc调用的功能
dexcloud-springboot-starter-web以打包的方式，提供了dexcloud-springboot-starter-jetty和dexcloud-springboot-starter-jersey的功能
dexcloud-springboot-starter-jms在springboot jms基础上进行了封装，支持与javaAF-mq的对接能力
dexcloud-springboot-starter-db-jdbc在springboot的DataSource功能上进行封装，提供了连接的分配跟踪的功能
dexcloud-springboot-starter-coordinator提供了基于zookeeper实现的分布式协调的功能
dexcloud-springboot-starter-stream-kafka提供了基于springcloud stream开发的kafka stream的功能
